## Findings

### #1 — CRITICAL: `movedToPosition` undefined in `handleReorder`

**Source:** Edge Case Hunter + Acceptance Auditor  
**Location:** `src/components/sop-editor.ts`  
**Category:** PATCH  

**Detail:**  
The variable `movedToPosition` is referenced in `handleReorder` but never declared. The diff shows:

```typescript
private handleReorder(ids: string[]) {
  const stepId = ids[0]; // First item is the moved step
  if (!stepId) return;
  const movedToPosition = ids.indexOf(stepId) + 1;  // ← NOT IN DIFF
  announce(`Step moved to position ${movedToPosition}`);
}
```

This will cause a runtime error when reordering steps. Additionally, the focus management for the moved step is mentioned in the spec (AC #6) but the diff excerpt does not show the focus code being executed.

### #2 — MAJOR: Typo breaks focus trap reverse shift logic

**Source:** Edge Case Hunter  
**Location:** `src/components/sop-screenshot-lightbox.ts`  
**Category:** PATCH  

**Detail:**  
`eShiftKey` is used instead of `e.shiftKey`. The second condition `!eShiftKey && document.activeElement === last` will always evaluate incorrectly because `eShiftKey` is undefined.

```typescript
} else if (!eShiftKey && document.activeElement === last) {  // BUG: eShiftKey undefined
  e.preventDefault();
  first.focus();
}
```

**Consequence:** Shift+Tab from last focusable element wraps to last instead of first, breaking expected keyboard navigation.

### #3 — MAJOR: Delete key triggers delete without confirmation

**Source:** Blind Hunter + Edge Case Hunter  
**Location:** `src/components/sop-step-card.ts`  
**Category:** PATCH  

**Detail:**  
`handleCardKeydown` dispatches delete on `Delete` key without user confirmation. Accidental keypress (especially for non-English keyboards where Delete may differ) permanently removes steps.

```typescript
} else if (e.key === 'Delete') {
  e.preventDefault();
  this.handleDelete();
}
```

The spec mentions AC #1 requires Delete key with undo, but the delete happens immediately before undo is available. Consider adding a confirmation or grace period.

### #4 — MAJOR: Rapid announcements can overwrite each other

**Source:** Edge Case Hunter + Blind Hunter  
**Location:** `src/components/sop-app.ts`  
**Category:** PATCH  

**Detail:**  
`announce()` sets textContent immediately with no queue or debounce. Screen readers may miss announcements if two state changes occur within one frame.

```typescript
export function announce(message: string, priority: 'polite' | 'assertive' = 'polite'): void {
  const el = document.getElementById('sop-announcer');
  if (!el) return;
  el.setAttribute('aria-live', priority);
  el.textContent = '';  // Clear first
  requestAnimationFrame(() => { el.textContent = message; });  // Set after rAF
}
```

**Consequence:** Screen reader may miss announcements (e.g., pause then resume within same frame).

### #5 — MAJOR: Focus lost when last step deleted (no fallback target)

**Source:** Edge Case Hunter  
**Location:** `src/components/sop-editor.ts`  
**Category:** PATCH  

**Detail:**  
When all steps are deleted, `cards.length === 0`, `focusIndex = Math.min(0, -1) = -1`, and the early return on `focusIndex >= 0` prevents crash. However, focus is lost entirely to the body.

```typescript
const focusIndex = Math.min(deletedIndex, cards.length - 1);
const card = focusIndex >= 0 ? cards[focusIndex] : undefined;
if (card) {
  const article = card.querySelector('article');
  (article as HTMLElement)?.focus();
}
```

**Consequence:** Keyboard users must navigate back to find focus.
