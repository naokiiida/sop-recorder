# Chrome Web Store Screenshot Specifications

Screenshots are NOT auto-generated — they must be manually captured from the built extension running in Chrome.

## Required Screenshots

### 1. Recording in Progress
- **Dimensions:** 1280x800 px
- **Format:** PNG or JPEG
- **Content:** Chrome browser with the nuknow side panel open, actively recording. The side panel should show the recording indicator (red dot), a few captured steps with thumbnails, and the target page visible in the main browser area.
- **Purpose:** Show the core recording experience.

### 2. Editing Steps
- **Dimensions:** 1280x800 px
- **Format:** PNG or JPEG
- **Content:** The SOP editor view with multiple steps displayed as cards. Show an expanded step with an edited description to highlight the editing capability. Include the step reorder and delete controls.
- **Purpose:** Demonstrate the editing and refinement workflow.

### 3. Exported SOP Result
- **Dimensions:** 1280x800 px
- **Format:** PNG or JPEG
- **Content:** A completed SOP viewed as rendered Markdown (e.g., in a GitHub README or Notion page). Show the numbered steps with annotated screenshot images and clear descriptions.
- **Purpose:** Show the polished final output users get from the tool.

### 4. Small Promo Tile
- **Dimensions:** 440x280 px
- **Format:** PNG
- **Content:** The nuknow logo/icon with tagline: "Record browser steps into shareable SOPs". Clean, minimal design with brand colors.
- **Purpose:** CWS promotional tile shown in search results and category pages.

## CWS Image Requirements Reference
- Screenshots: 1280x800 or 640x400, JPEG or PNG, max 5
- Small promo tile: 440x280, PNG
- All images must accurately represent the extension's functionality
